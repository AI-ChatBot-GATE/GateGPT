P12: ABSTRACT:
P13: Campus Bites is a full-stack smart canteen management system designed for university environments, built using React.js, FastAPI, and MongoDB with real-time order tracking via Socket.IO.
P14: The platform serves three roles — Students (browse, order, pay, track), Crew (manage live kitchen queue, validate tokens), and Management (menu, inventory, analytics, administration) — with JWT-based Role-Based Access Control securing all operations.
P15: Its core innovation lies in a multi-layered AI engine: the Food Recommendation module applies the Apriori algorithm on order history for real-time suggestions; the Wellness AI module uses fuzzy NLP to recommend health-appropriate foods based on a student's current condition; and the Meal Planner uses the 0/1 Knapsack algorithm to optimize meals against protein and budget goals. The system also simulates relational integrity (foreign key constraints, cascade updates, budget triggers) within MongoDB at the application layer.
P16: With 11 modules, 53 sub-modules, 9 master forms, 12 transaction forms, and 6 visualizations, Campus Bites delivers a comprehensive, production-ready solution integrating operational efficiency, AI personalization, and student wellness.
P17: Keywords: Canteen Management, FastAPI, React.js, MongoDB, Apriori, Knapsack, WebSockets, Wellness AI, RBAC

P100: How is your product different from existing products? 
P101: 1. Fitness-Aware Food Ordering
P102: The Difference: Swiggy and Zomato offer food ordering but have zero awareness of a user's health or fitness goals. MyFitnessPal tracks nutrition but has no connection to a real, live food vendor. These two worlds have never been merged in a campus context.
P103: The Execution: Campus Bites implements a Protein-Goal Integration Engine where every menu item in the canteen catalog carries a macro-nutrient profile (Protein, Carbs, Fats). When a student places an order, the system automatically syncs the meal's nutritional data to their Gym Freak fitness dashboard — making the canteen itself a measurable part of the student's health journey, not just a place to eat.
P104: 
P105: 2. Token-Validated Kitchen Handoff System
P106: The Difference: Swiggy tracks delivery riders, and McDonald's displays a token number on a static screen. Neither system actively validates or prevents a token from being claimed twice — a critical problem in high-traffic university canteens during peak lunch hours.
P107: The Execution: Campus Bites implements a Token Validation Engine inside the Crew module that verifies every pickup against a token_logs database record. A token can only be marked as "Handed Off" once, and the exact timestamp and crew member involved are permanently logged. This eliminates duplicate pickups, queue confusion, and disputes between students and kitchen staff entirely.
P108: 
P109: 3. Real-Time 3-Way Dashboard Synchronization
P110: The Difference: Existing canteen management tools are either student-facing (Swiggy) or kitchen-facing (POS systems). No platform currently delivers a simultaneous, live-synchronized view across the Student, Kitchen Crew, and System Admin — all three dashboards showing the same system state in real time.
P111: The Execution: Campus Bites uses a Socket.IO WebSocket backbone to achieve instant, three-way synchronization. When a Crew member updates an order to "Ready", the student's Order Tracking screen updates within milliseconds, and the Admin Revenue Dashboard logs the completed transaction simultaneously — with no page refresh, no polling, and no delay across any of the three dashboards.
P112: 
P113: 4. Apriori-Powered Contextual Combo Suggestions
P114: The Difference: Generic recommendation engines like Swiggy's "People Also Ordered" use popularity-based sorting, not true association mining. This produces generic, unhelpful suggestions that ignore the actual co-purchase patterns of the specific campus community.
P115: The Execution: Campus Bites implements the Apriori association rule mining algorithm directly on the canteen's own order history database. This means that if 78% of students at this specific campus who order Biryani also order a Coke, that exact insight is surfaced as a real-time suggestion — a data-proven, campus-specific combo recommendation that improves with every new order placed in the system.
