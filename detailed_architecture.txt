1. ARCHITECTURAL OVERVIEW
-------------------------
GateGPT is a decoupled, full-stack web application following the MERN-Next architecture 
pattern (Next.js, Node.js, Express, MongoDB). The system is designed for high 
responsiveness (Optimistic UI) and technical accuracy (AI Engineering).

2. CORE COMPONENTS
------------------

[ CLIENT-SIDE (FRONTEND) ] - Folder: /client
- Framework: Next.js 14 (App Router)
- UI Library: React 18
- Styling: Tailwind CSS & Framer Motion
- State Management: Zustand (Centralized stores for Auth, Chat, and Learning)
- Performance: Server-Side Rendering (SSR) for static content, Client-Side 
  Interactivity for the Chat and Analytics dashboards.
- Rendering: KaTeX for LaTeX/Math rendering and React-Markdown for technical content.

[ SERVER-SIDE (BACKEND) ] - Folder: /server
- Runtime: Node.js
- Framework: Express.js
- API Type: RESTful API
- Storage: MongoDB Atlas (Document-based NoSQL)
- AI Engine: @google/generative-ai (Interfacing with Gemini 3.1 Flash Lite)

3. DATA FLOW ARCHITECTURE
--------------------------

Phase 1: Request Initialization
   - User interacts with UI (e.g., sends a message or starts a practice quiz).
   - Frontend State (Zustand) updates optimistically to show immediate feedback.
   - Axios HTTP client sends the request to the /server with a JWT Bearer Token.

Phase 2: Backend Orchestration
   - Auth Middleware intercepts the request, validates the JWT, and attaches 
     the UserID to the request object.
   - The Controller receives the data and determines if it requires Database 
     access (Notes/Practice) or AI generation (Chat/Solver).

Phase 3: Intelligence Processing (Chat Flow)
   - Server constructs a "System-Grounded Prompt" for Gemini.
   - Gemini API processes the prompt and returns a technical response.
   - Server saves the message to the "Messages" collection in MongoDB.

Phase 4: Response & Update
   - Server sends the JSON response back to the client.
   - Frontend Zustand store updates the persistent state.
   - Recharts (if in Analytics) or KaTeX (if in Chat) renders the data.

4. SECURITY ARCHITECTURE
-------------------------

- Identity Management: JWT (JSON Web Tokens) for stateless authentication.
- Password Security: bcrypt.js for one-way salting and hashing of credentials.
- Endpoint Protection: Custom middleware `authMiddleware.js` protecting all 
  `/protected` and `/api` routes from unauthorized access.
- Cross-Origin Safety: CORS configuration limiting requests only from the 
  specified FRONTEND_URL.

5. DATABASE SCHEMA (MONGODB)
-----------------------------

- Users Collection: { _id, email, password, profile: { name, targetRank } }
- Chats Collection: { _id, userId, title, isPinned, isArchived, messages: [Ref] }
- Messages Collection: { _id, chatId, role, content, createdAt }
- PracticeResults: { _id, userId, subject, topic, score, timeTaken, difficulty }
- Notes Table: { _id, userId, subject, topic, content, summary, tags: [] }

6. AI PROMPT ENGINEERING LAYER
-------------------------------

The system utilizes "Structured Instruction Sets" to ensure technical accuracy:
- STEM Grounding: Prompts instruct the model to use formal Engineering notation.
- Step-by-Step Chain: Forced reasoning steps before providing final numerical answers.
- Format Enforcement: Requirements to output in Markdown/LaTeX compatible formats.

7. FRONTEND ROUTING SYSTEM (NEXT.JS)
------------------------------------

/app
  /(auth)         -> Login, Signup (Public)
  /(protected)    -> Chat, Analytics, Subjects, Notes (Requires JWT)
    /dashboard
    /analytics
    /practice
/lib
  axios.ts        -> Base configuration for API communication.
/store
  useChatStore.ts -> State logic for conversational data.

8. INTEGRATION & LIBRARIES
---------------------------
- Recharts: Horizontal Bar Charts for mastery and Area Charts for accuracy trends.
- Lucide-React: Icon set for high-end aesthetic.
- KaTeX: Parsing TeX strings into high-quality math displays.
- Mongoose: ODM for modeling MongoDB data as JavaScript objects.
